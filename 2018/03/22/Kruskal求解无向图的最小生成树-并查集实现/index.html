发表于 2018-03-22 |

Kruskal 概述

Kruskal算法，是用来求加权连通图的最小生成树的算法。

它的基本思想是：按照权值从小到大的顺序选择n-1条边，并保证这n-1条边不构成回路（环）。

具体做法是：首先构造一个只含n个顶点的森林，
然后依权值从小到大从连通网中选择边加入到森林中，
并使森林中不产生回路，直至森林变成一棵树为止。

更多关于kruskal的知识欢迎查看维基百科-Kruskal’s algorithm

并查集（union-find set）概述

union-find set 翻译成中文就是并查集,
并查集是一种树型的数据结构，用于处理一些不相交集合（disjoint sets）的合并及查询问题。

并查集的主要操作：

1. 初始化：把每个点所在集合初始化为其自身。

2. 查找：查找元素所在的集合，即根节点。

3. 合并：将两个元素所在的集合合并为一个集合。

并查集判断无向图是否有环

如上图, 如何判断该图是否有环呢？首先你可能会想到用递归的方式去做所以你用了DFS，
没错DFS确实可以做到，但是复杂度过高，那么能否用并查集来判断无向图是否有环呢？
答案是肯定的。

首先规定如果两个点之间有边相连，如果这两个点不属于同一个集合，那就将他们合并成一个集合

  1. 我们把每个点看成独立的集合{0} ，{1}， {2}
  2. 对于边0-1，直接将这两个点代表的集合合并{0， 1}, 让1来当父节点
  3. 对于边1-2， 它们分别属于不同的集合，合并集合之后是{1，2}，让2来当父节点，
    这时候有0->1->2 即 0的祖先节点是2
  4. 对于边0-2，他们属于一个集合，因为他们有着共同的祖先2，
    这就说明0-2之间在没有0-2这条边之前已经连通了，
    如果加上这条边的话那从0到2就有两条路径可达，就说明存在环了
