NLP中的特征提取及特征选择

发表于 2019-03-24 |

本文主要记录 NLP 中的特征提取以及特征选择，会借助 sklearn 工具给出具体的例子

预备知识

文本向量化

“向量化”可以理解为“数值化”，为什么要“数值化”？因为文字是不可以运算的，而数值可以。现今的 NLP 方法主要基于统计的方法所以向量化往往是建模的第一步；但也有的方法是基于规则的，这些不用向量化，一般是基于正则匹配的方法完成。

向量化的常用方式

传统向量表示

  • 独热编码 one-hot
  • 词集模型
  • 词袋模型
  • TF-IDF

分布式向量表示

  • word2vec / fasttext / glove (词级别)
  • ELMo / GPT / BERT (句子级别)

特征提取

NLP中字、词、词频、ngram、词性…等都可以认为是特征，只要将这些特征向量化就能放入模型中计算。比如词袋模型就是利用的词频特征，word2vec 可以认为是利用了窗口内文本的共现特征。

使用 sklearn 提取特征

TfidfVectorizer

提取 TF-IDF 特征

CBOW 更多利用的是词频信息，而 tf-idf 可以把词频和文档频率特征利用起来，在文本分类、聚类、以及文本检索排序上有较多的应用。关于 tf-idf 可以查阅本人写的另一篇文章为朴素贝叶斯加入tf-idf特征 
sklearn 封装的 TfidfVectorizer 和 CountVectorizer 的使用一样，大部分参数也相同，详细参数见文档 

输出

DictVectorizer

DictVectorizer 可以从 python 的 dict 中得到向量化表示，对于 dict 中的数值型元素直接以原始数值表示，非数值型会转换为 one-hot 编码表征。一般对于 表格 类型的数据如 csv , excel 等文件类型的数据我们都可以转成 dict 然后使用 DictVectorizer 向量化。更多关于 DictVectorizer 的信息，详见 文档  

输出

HashingVectorizer

对于越大的语料库，词汇量往往也越大，传统的 VSM 方式得到的向量是高维稀疏的这会使用更多内存。为了解决内存不够用的问题，我们一般有两种方式处理：1. 哈希 2. 使用分布式向量（后面会介绍）在此主要讲解 sklearn 提供的哈希向量化。和之前的向量化类似，HashingVectorizer 提供的类似的接口，类似的参数，它的参数详见文档 

输出

分布式向量的提取

在深度学习方法解决 NLP 问题中 Embedding 往往是网络的第一层，Embedding 又称 嵌入 它可以把特征表示成分布式向量，如果特征是词我们可以成为词嵌入(word embedding)，字可以成为字嵌入(char embedding)。常用的嵌入算法有 word2vec 、fasttext 、glove 等。由于深度学习方法往往训练周期比较长这导致了它的灵活性不强，在真正的生产模型中使用更为灵活的线性模型更多。我们的兴趣小组 4AI 写了一个简单的库可以生成分布式向量表示然后放进 sklearn 封装的线性模型中训练，这个库主要基于 gensim 提供的 word2vec 、 fasttext 实现的，库的地址： Github: embedding_features
